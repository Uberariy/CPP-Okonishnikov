1. Описание наборов входных пакетов (l2sw_in.pcap):

	1: Пакет с SRC MAC равным ff:ff:ff:ff:ff:ff:ff. Он будет сброшен, т.к этот адрес недействителен.
	2: Пакет, чей ключ не возимеет результата поиска при поиске по нему в таблице классификации 2. Этот пакет будет отправлен на порт управления (L2 коммутация 9).
	3-4: Пакеты, которые найдут соответствие своим ключам в таблице классификации 2. 8 младших битов VLAN ID будут изменены на значение равное результату поиска (L2 коммутация 7-8).
	5: Пакет с неизвестным SRC MAC, он будет отправлен на порт управления (L2 коммутация 6).
	6: Пакет, имеющий широковещательный DST MAC. Он будет отправлен всем портам, кроме порта управления (L2 коммутация 3-5).
	7: Пакет, не содержащий VLAN TAG. Такой пакет будет сброшен (L2 коммутация 1).


2. Описание ключей и результатов поиска (l2sw_tables.json):

	КЛЮЧ:			РЕЗУЛЬТАТ:
	1:
	010203040506		01		 
	/\ - "Известный" SRC MAC. Пакеты, содержащие иные SRC MAC (Например, пакет 5 в l2sw_in.pcap) будут направлены на порт управления.

	2:
	00030303030381000001	08		
	00040404040481000001	10		
	/\ - 10-байтовые ключи формата DSTMAC:VLANTAG; Результат поиска это номер порта, на который пакеты с таким ключам будут направлен (В файле l2sw_in.pcap эта возможность представлена пакетами 3 и 4. Данные пакеты будут направлены на порты 3 и 4, соответственно)
