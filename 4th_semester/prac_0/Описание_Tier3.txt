1. Описание наборов входных пакетов (l2sw_in3.pcap):

	1-7: =||= (Обработка ARP запросов 1)
	8. Пакет, имеющий номер канального протокола передачи данных, несовпадающий с номером Ethernet (0x0001), он будет сброшен (Обработка APR запросов 2).
	9. Пакет, содержащий ARP ответ, будет отправлен широковещательно (Обработка ARP запросов 4).
	10. Пакет, содержащий ARP запрос, его Target Protocol Address содержится в таблице => Его Opcode, а также Target Hardware Address будут изменены (Обработка APR запросов 5-6).
	11. Пакет, содержащий ARP запрос. Однако его Target Protocol Address не содержится в таблице. Он будет отправлен на порт управления. (Обработка APR запросов 5).

В качестве протокольных адресов у ARP протокола может быть только IP (источник: http://www.networksorcery.com/enp/protocol/arp.htm#Protocol%20type), поэтому пакет, имеющий иной протокольный адрес не представлен (Обработка ARP запросов 3).
